  <h1><a name="SECTION004500000000000000000">&nbsp;</a> <br> 2.5 Troubleshooting </h1> 
  <p> There are a few things you can try to identify the problem: </p>
  <p> </p>
  <ul> 
   <li>Carefully study the error output, if any. <p> </p></li> 
   <li>Check the server error log file, it may contain useful clues. <p> </p></li> 
   <li>Try running Apache from the command line in single process mode: 
    <div class="verbatim">
     <pre>
    ./httpd -X
</pre>
    </div> This prevents it from backgrounding itself and may provide some useful information. <p> </p></li> 
   <li>Beginning with mod_python 3.2.0, you can use the mod_python.testhandler to diagnose your configuration. Add this to your <span class="file">httpd.conf</span> file : <p> </p>
    <div class="verbatim">
     <pre>
    &lt;Location /mpinfo&gt;
      SetHandler mod_python
      PythonHandler mod_python.testhandler
    &lt;/Location&gt;
</pre>
    </div> <p> Now point your browser to the <span class="file">/mpinfo</span> URL (e.g. <span class="file">http://localhost/mpinfo</span>) and note down the information given. This will help you reporting your problem to the mod_python list. </p><p> </p></li> 
   <li>Ask on the mod_python list. Make sure to provide specifics such as: <p> </p>
    <ul> 
     <li>Mod_python version. </li> 
     <li>Your operating system type, name and version. </li> 
     <li>Your Python version, and any unusual compilation options. </li> 
     <li>Your Apache version. </li> 
     <li>Relevant parts of the Apache config, .htaccess. </li> 
     <li>Relevant parts of the Python code. <p> </p></li> 
    </ul> <p> </p></li> 
  </ul>
