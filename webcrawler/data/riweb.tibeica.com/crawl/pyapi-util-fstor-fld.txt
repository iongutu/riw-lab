  <h2><a name="SECTION006620000000000000000">&nbsp;</a> <br> 4.6.2 Field class </h2> 
  <p> </p>
  <dl>
   <dt>
    <table cellpadding="0" cellspacing="0">
     <tbody>
      <tr valign="baseline"> 
       <td>
        <nobr>
         <b><span class="typelabel">class</span>&nbsp;<a name="l2h-187"><tt class="class">Field</tt></a></b>(
        </nobr></td> 
       <td>)</td>
      </tr>
     </tbody>
    </table> 
   </dt>
   <dd>
     This class is used internally by <tt class="class">FieldStorage</tt> and is not meant to be instantiated by the user. Each instance of a <tt class="class">Field</tt> class represents an HTML Form input. 
    <p> <tt class="class">Field</tt> instances have the following attributes: </p>
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-188"><tt class="member">name</tt></a></b> 
     </dt>
     <dd>
       The input name. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-189"><tt class="member">value</tt></a></b> 
     </dt>
     <dd>
       The input value. This attribute can be used to read data from a file upload as well, but one has to exercise caution when dealing with large files since when accessed via <tt class="member">value</tt>, the whole file is read into memory. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-190"><tt class="member">file</tt></a></b> 
     </dt>
     <dd>
       This is a file-like object. For file uploads it points to a <tt class="class">TemporaryFile</tt> instance. (For more information see the TemporaryFile class in the standard python <em class="citetitle"><a href="http://docs.python.org/lib/module-tempfile.html" title="tempfile">tempfile</a></em> module). 
      <p> For simple values, it is a <tt class="class">StringIO</tt> object, so you can read simple string values via this attribute instead of using the <tt class="member">value</tt> attribute as well. </p>
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-191"><tt class="member">filename</tt></a></b> 
     </dt>
     <dd>
       The name of the file as provided by the client. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-192"><tt class="member">type</tt></a></b> 
     </dt>
     <dd>
       The content-type for this input as provided by the client. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-193"><tt class="member">type_options</tt></a></b> 
     </dt>
     <dd>
       This is what follows the actual content type in the <code>content-type</code> header provided by the client, if anything. This is a dictionary. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-194"><tt class="member">disposition</tt></a></b> 
     </dt>
     <dd>
       The value of the first part of the <code>content-disposition</code> header. 
     </dd>
    </dl> 
    <p> </p>
    <dl>
     <dt>
      <b><a name="l2h-195"><tt class="member">disposition_options</tt></a></b> 
     </dt>
     <dd>
       The second part (if any) of the <code>content-disposition</code> header in the form of a dictionary. 
     </dd>
    </dl> 
    <p> </p>

       for a description of form-based file uploads
